**Copywriting rules:**
- Toast title: max 60 characters. Identify the problem.
- Toast description: max 120 characters. State what the user can do next.
- Do not use "Fehler" as the toast title for auto-save — reserve that for hard blocking errors.
  Use "konnte nicht gespeichert werden" to signal a recoverable background failure.

## Interaction Contract

This section covers the behavioral contract for new interactions introduced in this phase.
Executor must implement these exactly.

### Auto-Save on Dialog Close

Trigger: `onOpenChange` fires `false` (X button, backdrop click, or "Abbrechen" button).

Gate: `selectedCustomer !== null` (D-03). If gate fails, skip save and call `onClose()`.

Behavior:
1. If `draftId` ref is null → call `invoicesApi.createDraft(formData)` → store returned `id` in `draftId` ref.
2. If `draftId` ref has a value → call `invoicesApi.updateDraft(draftId, formData)`.
3. On success: call `onClose()`. Silent — no toast, no spinner visible to user.
4. On failure: show destructive toast (see Copywriting Contract above), then call `onClose()`.
   Do NOT block close on save failure.

### Auto-Save on Navigation (useBlocker)

Trigger: React Router `useBlocker` detects pending navigation while dialog `open === true`.

Behavior:
1. Run the same gate check (D-03).
2. If gate passes: fire save (same createDraft/updateDraft logic as above).
3. Proceed with navigation immediately — do NOT await or block navigation on result (D-05).
4. On failure: destructive toast fires after navigation completes (best-effort, may not render).
   Accept this limitation for Phase 2. Phase 4 adds a reliable indicator.

### Finalize Flow (Draft → Real Invoice)

Trigger: User submits form via "Rechnung erstellen".

Behavior:
1. Call existing `POST /invoices` create path with full form data (D-07).
2. If `draftId` ref has a value: call `invoicesApi.deleteDraft(draftId)` as best-effort (D-08).
   - Do not await the delete before showing success toast.
   - If delete fails: log to console, do not toast, do not block user.
3. Show existing success toast: "Rechnung erstellt".

### No Loading Indicator

Phase 2 does NOT add a saving spinner, "Wird gespeichert..." label, or any visual saving
indicator to the dialog close button or dialog header. That visual affordance is deferred
to Phase 4. Executor must not add one.
