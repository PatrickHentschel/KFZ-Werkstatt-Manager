<objective>
Relax DB-level NOT NULL constraints on `invoices.customer_id` and `invoices.issue_date` so that draft invoices can be persisted with partial data per locked decision D-04 ("No fields are required to create a draft").

Purpose: This is a hard prerequisite for the upsert endpoint (Plan 02). Without nullable columns, `createDraft()` cannot honour D-04 and would either reject empty payloads (violating D-04) or require sentinel rows (violating FK integrity). The user has explicitly approved the migration path (resolving the schema conflict from RESEARCH.md "Schema Constraint Reconciliation").

Output:
- Modified Drizzle schema file (`backend/src/db/schema/invoices.ts`)
- One new generated SQL migration file under `backend/src/db/migrations/`
- Migration applied to local Postgres (Docker)
</objective>

| T-01-04 | Denial of Service | `ALTER TABLE` takes an `ACCESS EXCLUSIVE` lock on `invoices` and blocks live queries during a long migration | accept | This is a DDL metadata-only change (drop NOT NULL); Postgres performs it in O(1) with a brief lock.
